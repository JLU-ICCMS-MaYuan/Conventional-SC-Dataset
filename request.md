# 用户需求记录

## 需求 #1 - 管理员系统和文献审核功能

**提交时间**: 2025-11-30

### 核心需求

1. **文献审核状态**
   - 每篇上传的文献都要标记是否被管理员检查过
   - 审核状态：未审核 / 已审核（通过） / 已拒绝 / 待修改
   - 记录审核人（管理员邮箱）和审核时间
   - **重要**：文献无论是否审核都可以上传和显示

2. **管理员系统**
   - 注册方式：通过邮箱注册
   - 验证方式：SMTP 邮箱验证码（163邮箱）
   - 审批制度：超级管理员需要批准新管理员申请
   - 实名要求：管理员需要提供真实姓名
   - 审核记录：标记每篇文献被哪个管理员（邮箱）检查

3. **用户体验**
   - 保持开放贡献：任何人都可以匿名上传，标记为"未审核"
   - 列表显示：显示审核状态徽章，可筛选
   - 详情页面：显示审核人信息

### 技术选型确认

- **审核状态含义**: 质量认证（通过/拒绝/待修改）
- **匿名上传策略**: 完全匿名（无需留邮箱）
- **超级管理员初始化**: 通过环境变量 `SUPERADMIN_EMAIL` 指定
- **SMTP 服务商**: 163邮箱 (smtp.163.com)

### 架构问题警告

⚠️ **已识别的潜在问题**:
1. 完全匿名上传 + 质量认证审核 = 无法联系上传者修改被拒绝的文献
2. 可能导致大量"僵尸文献"堆积（状态为"已拒绝"但无人修改）
3. 建议：至少让上传者**可选**留下邮箱，方便管理员沟通
